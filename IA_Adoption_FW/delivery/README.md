# Framework IA for Delivery

> Intégration IA en équipe - Processus collectifs et livraison

## Pour qui ?

**Équipes et leads techniques** qui veulent intégrer l'IA dans leurs processus : code review, documentation, testing, refactoring.

**Prérequis** : Niveaux 2-3 (compétences intermédiaires à avancées) - maîtrise outils personnels, prompt engineering, RAG/ML basics. Voir [GLOSSARY.md](../GLOSSARY.md).

## Structure

6 phases : **Strategy** → **Plan** → **Ready** → **Govern** → **Secure** → **Manage**

| Phase | Focus | Timeline |
|-------|-------|----------|
| [Strategy](01-strategy.md) | Explorer patterns équipe | 2-4 semaines |
| [Plan](02-plan.md) | Planifier adoption | 4-6 semaines |
| [Ready](03-ready.md) | Infra équipe | 1-2 semaines |
| [Govern](04-govern.md) | Politiques (5-10 pages) | 1 semaine |
| [Secure](05-secure.md) | OWASP + DLP basics | 1-2 semaines |
| [Manage](06-manage.md) | MLOps basique | Ongoing |

**Caractéristiques** : Gouvernance MEDIUM, Sécurité PRAGMATIQUE, Coût ~k€/mois, Risque MEDIUM

## Quick Start

1. Vérifier prérequis (Niveau 2+)
2. Explorer patterns d'équipe
3. Choisir 1-2 patterns à tester (2 semaines)
4. Établir politiques équipe
5. Implémenter contrôles sécurité
6. Mesurer impact

**Progression** : Niveau 3 → [Framework Automation](../automation/README.md)
